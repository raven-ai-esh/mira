# MIRA Public Non-Goals

Date: `2026-03-19`
Baseline: `MIRA 2.6.0`

This page exists to keep public positioning honest.

## What MIRA Is Not

`MIRA` is not currently presented as:

- a universal replacement for `Python`
- a universal replacement for `Go`
- a universal replacement for `Rust`
- a frontend platform
- a browser/UI language
- a human-first language optimized for syntax sugar
- a claim of performance dominance outside benchmarked promoted workloads

## What We Do Not Claim

We do not currently claim:

- that `MIRA` is best for every backend workload
- that `MIRA` should replace general scripting languages for all glue code
- that `MIRA` has frontend or fullstack parity
- that every runtime surface outside the documented promoted scope is equally
  mature
- that benchmark wins generalize beyond the workloads captured in release
  evidence

## Why These Non-Goals Matter

`MIRA` grows through trust, not through vague reach.

The public thesis stays narrow on purpose:

- stronger claims would be easier to say
- stronger claims would also be easier to disprove

The platform is more useful if outsiders can quickly see:

- what they can use today
- what is still outside scope
- which artifacts prove the current claim

## Current Contribution Guardrail

Until the dedicated community-contribution release lands, public contribution
policy stays narrow:

- contributions must come with reproducible evidence or concrete reviewable
  artifacts
- benchmark claims must link to benchmark artifacts
- example claims must link to runnable source or proof artifacts
- maintainers may reject off-scope or hype-driven additions even when they are
  enthusiastic

## Current Positive Scope

As of `2.6.0`, public-positive scope is:

- agent-first backend services
- promoted messaging workloads
- promoted analytics/statistics workloads
- promoted distributed backend coordination paths already covered by release
  evidence
