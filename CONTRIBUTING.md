# MIRA Public Contributing Guide

Date: `2026-03-19`
Status: `public repo contribution entry`

This is the public contribution baseline for `MIRA`.

## What Contributions Are Useful Right Now

- reproducible onboarding friction reports
- docs clarity fixes
- benchmark reproduction reports
- verification mismatches between docs and artifacts
- small example improvements inside released workload scopes

## What To Include In A Good Issue

- exact document or example path
- exact command or sequence you ran
- actual output
- expected output
- whether the problem is about docs, verification, performance, or claims scope

## Contribution Rules

- keep claims scoped to released evidence
- do not broaden benchmark claims without artifact-backed proof
- do not claim universal replacement of `Python`, `Go`, or `Rust`
- prefer small, reproducible reports over vague feedback
